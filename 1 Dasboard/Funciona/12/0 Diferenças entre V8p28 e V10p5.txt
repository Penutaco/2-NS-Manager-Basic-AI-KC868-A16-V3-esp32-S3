Análise de Riscos das Lacunas de Código na V10p5
🔴 RISCOS CRÍTICOS (Programa para completamente):
detect_prgsample_ph_measurement() - RISCO ALTO
O que faz: Processa valores pH/EC do event_data.csv
Lacuna: try/except incompleto para parsing de valores
Risco: CRASH quando encontra dados malformados
Sintoma: ValueError ou IndexError → programa termina
Impacto: Sistema não consegue ler sensores → ZERO dosagem
send_dosing_command() - RISCO ALTO
O que faz: Envia comandos para ativar bombas
Lacuna: Response handling incompleto
Risco: TIMEOUT ou resposta malformada → programa trava
Sintoma: ZMQ socket fica bloqueado → todas as dosagens falham
Impacto: Plantas morrem por falta de correção pH/EC
🟡 RISCOS MODERADOS (Funcionalidade degradada):

O que faz: Adiciona eventos ao buffer para sincronização
Lacuna: Código para adicionar ao buffer
Risco: Eventos #6 perdidos → dosagem não sincronizada
Sintoma: "Event #6 not detected" repetidamente
Impacto: Dosagem aleatória → overdose ou underdose
get_current_cycle_id() - RISCO MÉDIO
O que faz: Extrai cycle ID do ficheiro para tracking
Lacuna: Fallback logic e processamento de linhas
Risco: IDs incorretos → dados de aprendizagem corrompidos
Sintoma: Coefficients não atualizados corretamente
Impacto: Dosagem progressivamente pior
chemistry_based_dosing_decision() - RISCO MÉDIO
O que faz: Calcula volumes de dosagem
Lacuna: return statement na função pH-
Risco: pH- nunca retorna dosagem → pH nunca baixa
Sintoma: pH fica sempre alto → plantas stressadas
Impacto: Crescimento deficiente
🟢 RISCOS BAIXOS (Degradação graceful):

Lacuna: Loop de processamento de dados

detect_prg_events() - RISCO BAIXO

Risco: Não detecta calibrações → dosa durante calibração

check_event_buffer_for_event6() - RISCO BAIXO
